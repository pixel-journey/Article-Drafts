# Content Repurposing Guide: From Docs to Articles to Social Empire

**Core Philosophy:** One piece of high-quality research/writing fuels 10+ assets across channels and time. This is how we maintain consistent presence without burning out.

## The Repurposing Funnel

1. **Source Material** (Docs, gitbook, team notes, vision docs, PxLabs specs)
   → Research & verify in `notes-and-sources.md`

2. **Full Article Draft** (`full-article-draft.md`)
   → Narrative, engaging long-form (Medium-ready, PxLearn module ready, website blog)

3. **X Adaptations**
   - `x-initial-announcement-post.md` (launch thread or single post + image)
   - `x-thread-version.md` (standalone thread that can live on its own)
   - `recurring-x-posts.md` (evergreen snippets for weeks/months/years later)

4. **Visual Layer** (`image-graphics-prompts.md`)
   → Generate hero images, carousels (3-6 slides), infographics, meme variations using Grok Imagine
   → Store generated images in artifacts or dedicated assets repo (future)

5. **Other Formats & Channels**
   - Discord long posts / threads
   - Email newsletter excerpts or full feature
   - Carousel image series (text from x-thread or dedicated)
   - Short video scripts / Reels ideas (future, using ffmpeg skill?)
   - PDF one-pagers or cheat sheets (pdf skill)
   - DOCX editable versions for team collab (docx skill)

## Example Flow for One Article

**Topic:** "What is the Pixel Journey?"

- Day 0: Research from llms.txt + the-pixel-journey-begins.md + pxportals-and-pxpackages.md
- Day 1: Write full-article-draft.md (1200 words)
- Day 2: Create x-initial + x-thread + 8 recurring ideas in recurring-x-posts.md + image prompts
- Day 3: Generate 2-3 hero/carousel images
- Day 4: Post initial on X + schedule 2-3 recurring for next weeks
- Ongoing: Update notes when new milestones happen; refresh visuals or add new fun facts

## Benefits

- Maximizes ROI on research/writing time
- Keeps @PxJourney feed active and valuable even on quiet development days
- Builds SEO authority (Medium + docs cross-links)
- Creates "content moat" — competitors can't easily replicate the depth + fun tone
- Easy onboarding for new contributors (templates + this guide)

## Tracking & Maintenance

- Use a simple table or Notion/Airtable (future) to track: Article # | Status (Draft / X Assets / Posted / Evergreen) | Last Refreshed | Performance Notes
- Every 3-6 months, review top performers and create updated "v2" versions with new data
- Tag articles that pair well for series or bundles (e.g. Onboarding series, Portal launch series)

This system turns our documentation strength into a living, breathing content engine that educates the world about the Pixel Journey while growing our community.
