# Article-Drafts

**Pixel Journey Organization**

Repository for drafting, planning, refining, and preparing high-quality articles, social content, and promotional materials for the PxJourney project.

Each folder represents one article and contains the full draft plus supporting assets (social posts, fun facts, key points, etc.).

## Purpose

This is the active working space where we:
- Draft and iterate on articles
- Audit and refine content to the highest standards
- Ensure consistent **Px** branding
- Align articles with official documentation at https://docs.pixeljourney.xyz
- Maintain factual accuracy with the current state of all Px portals and packages

## Core Principles

- **Pixel-perfect quality**: We continuously raise standards with each iteration.
- **Safety first**: All tracking and flagging systems are non-destructive. We never risk or delete existing work.
- **Alignment**: Articles, official docs, X content (@PxJourney), and actual built features must stay consistent.
- **Px branding discipline**: Use "Px" correctly for the journey and products. "Pixals" is reserved exclusively for the PFP collection.

## Key Living Documents

These files guide our work and should be referenced often:

- `PXJOURNEY_IMPROVEMENT_ROADMAP.md` — Central prioritized plan for all improvements across articles, docs, and processes.
- `ARTICLE_AUDIT_TRACKER.md` — Safe tracker for reviewing the first ~40+ articles (flags issues only, does not modify content).
- `PX_BRANDING_TERMINOLOGY_AND_AUDIT_FRAMEWORK.md` — Branding rules and audit guidelines.
- `FIRST_40_ARTICLES_AUDIT_AND_LINKING_FRAMEWORK.md` — How to audit content and add proper links to docs.pixeljourney.xyz.

## How We Work

1. New article drafts are created in their own folders.
2. We use the frameworks above to review and refine.
3. Issues are flagged in the tracker or via issues (never by directly editing without review).
4. We iterate and raise quality standards with each pass.
5. Once an article meets the current high standard, it moves toward publication.
